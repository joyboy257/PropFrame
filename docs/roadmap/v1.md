# Roadmap — v1

Target: working end-to-end prototype that a real estate agent can use.

## Must have

### Core loop
- [x] Project CRUD
- [x] Photo upload (browser → R2 presigned URL)
- [x] Clip generation request (queue job)
- [x] GPU worker processing (Ken Burns ffmpeg)
- [x] Clip status polling + download
- [ ] **AI video generation** — replace Ken Burns with actual video model
- [ ] **Virtual staging** — AI describing/transforming rooms
- [ ] **Sky replacement** — AI sky swap for exterior photos

### Payments
- [x] Stripe checkout
- [x] Credits balance display
- [x] Webhook → credit grant
- [ ] **Credit deduction on clip generation** (debiting not wired to worker yet)

### Auth
- [x] Magic link login
- [x] JWT middleware on protected routes
- [ ] **Account creation** (email + password fallback)
- [ ] **Magic link rate limiting**

### Auto-edit
- [ ] **Title text generation** (Groq → clip titles)
- [ ] **Clip ordering + assembly** (ffmpeg concat)
- [ ] **AI music generation** (HeartMuLa or API)
- [ ] **Final MP4 render** (worker assembles + uploads)

## Nice to have

- [ ] Project thumbnails (first photo)
- [ ] Email notification when clip is done
- [ ] Project rename/delete
- [ ] Batch clip generation (all photos → all clips)
- [ ] Resolution selector (720p/1080p/4k)
- [ ] Motion style selector UI (8 Ken Burns styles)

## Out of scope for v1

- Mobile app (iOS React Native)
- Team collaboration
- MLS integration
- White labeling
- Webhooks for third-party integrations

## Blockers

1. **AI video model** — the Ken Burns ffmpeg version is a placeholder. Need to pick a video generation provider.
2. **Credit deduction** — the worker generates clips but doesn't debit the user's balance yet.
3. **Auto-edit assembly** — no worker code for stitching clips + music together.

## Next session priorities

1. Wire AI text (Groq) into the worker for title generation
2. Build auto-edit assembly pipeline (ffmpeg concat + music)
3. Test the full clip → download loop end-to-end
